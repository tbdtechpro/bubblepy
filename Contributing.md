# Contributing

## What this project is

This is an AI-assisted experiment, not a community-maintained library. The Python code,
commit history, and documentation are primarily generated by Claude (Anthropic). The
human's role is experimental design, direction, and review — not direct authorship.

Contributions are welcome, but be aware that:

- The Python port is alpha (v0.1.0) and unvalidated.
- Many Go features are not yet ported. See [`MVP_TASKS.md`](MVP_TASKS.md) for the current
  gap list and [`PYTHON_FEASIBILITY.md`](PYTHON_FEASIBILITY.md) for features that are
  structurally difficult to replicate in Python.
- 117 tests exist across `keys`, `mouse`, `commands`, `renderer`, `screen`, and `Program`
  lifecycle. Run them with `pytest`.

## Development setup

```bash
git clone <repo-url>
cd bubbletea
python -m venv venv
source venv/bin/activate          # or venv\Scripts\activate on Windows
pip install -e ".[dev]"
```

---

## Running checks

```bash
# Tests
pytest

# Type checking
mypy .

# Formatting
black --check .
isort --check .

# Linting
flake8
```

Formatters can be run without `--check` to apply changes in place:

```bash
black .
isort .
```

---

## Go development

The Go library at the root is the upstream Charm implementation. Go tests and linting
run via [Task](https://taskfile.dev):

```bash
task test   # go test ./...
task lint   # golangci-lint run
```

## Picking up a task

Open tasks are tracked in [`MVP_TASKS.md`](MVP_TASKS.md). Unchecked items are fair game.
If you're implementing something non-trivial, check `PYTHON_FEASIBILITY.md` first — some
Go features have fundamental constraints in Python that affect the right approach.
